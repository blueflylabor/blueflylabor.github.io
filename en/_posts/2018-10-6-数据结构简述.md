# Introduction to Data Structures
## I. Data Structure: A Collection of Elements with One or More Specific Relationships
- Structure: Any problem, data elements do not exist independently, and there are relationships between them
- Logical structure
- Storage structure (physical structure)
- Data operations
- The logical structure and storage structure are inseparable
- Algorithm design depends on the logical structure, and implementation relies on the storage structure
## II. Logical Structure: Logical Relationships Between Data Elements
- Independent of storage, independent of the computer
- Divided into linear structures and non-linear structures
Linear structures: Linear table, stack, queue, string, array, generalized list
Non-linear structures: Tree, binary tree, directed graph, undirected graph
## III. Storage Structure (Physical Structure): The Representation of Data Structures in a Computer
- Representation of data structure
- Representation of relationships
- Dependent on the computer language
- Divided into sequential storage, linked storage, indexed storage, and hash storage
### 1. Sequential Storage: Physical storage locations are adjacent, and logically adjacent elements have adjacent physical locations
- Pros: Can achieve random access, each element occupies a small amount of memory
- Cons: Can only use one contiguous block of storage, resulting in significant external fragmentation
### 2. Linked Storage: The physical location of the stored data may not be adjacent, and logically adjacent elements may not have adjacent physical locations. Adjacent elements are found by storing the physical addresses of the adjacent elements
- Pros: No fragmentation, fully utilizing storage units
- Cons: Can only be stored sequentially
### 3. Indexed Storage: Similar to a catalog
### 4. Hash Storage: Calculates the physical address of the element directly based on the keyword

## V. Five Characteristics of Algorithms
- Finiteness: Execute a finite number of steps and end
- Determinism: Each instruction has a definite meaning, and the same input produces the same output
- Feasibility: Through basic operations implemented, obtain a definite result in a finite number of times
- Input: Zero or multiple inputs
- Output: One or more program outputs
## VI. Algorithm Complexity
### 1. Time Complexity: Measures how quickly the execution time of an algorithm increases as the problem size increases
### 2. Space Complexity: Measures how quickly the space occupied by an algorithm increases as the problem size increases
