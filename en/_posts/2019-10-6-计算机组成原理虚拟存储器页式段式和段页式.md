# Virtual Memory · Paging · Segmentation · Segmented Paging
Main memory and secondary storage together form virtual memory, working under the joint management of hardware and system software. For programmers, virtual memory is transparent, unifying the addressing space of main memory and secondary storage.
Users can use addresses involved in programming, which are called virtual or logical addresses, to access a storage area, which is called virtual space.
The actual address unit in main memory is called physical address or real address, and the corresponding storage area is called main memory address space or real address space.
- Physical Address = Page Number + Offset within Page
- Virtual Address = Virtual Page Number + Offset within Page
- Secondary Storage Address = Disk Number + Track Number + Cylinder Number + Sector Number

The CPU uses virtual addresses to access memory, and the auxiliary hardware finds the corresponding relationship between the virtual and physical addresses, and determines whether this virtual address corresponds to a storage unit that is currently in memory.
- In memory, the CPU can directly access data units indicated by the address through address translation.
- If not in memory, the page or segment containing the character is called into main memory, and then accessed through virtual and physical address transformation.

## Paging Virtual Memory
Using pages as the basic unit, the virtual space and main memory space are divided into equally sized pages. The page in main memory is called a **physical page** or **frame**, and the page in secondary storage is called a **virtual page**.
### Page Table
A page table is a table stored in main memory that records the correspondence between virtual page numbers and physical page numbers, recording the location of the virtual page in main memory when the program is loaded. The page table typically exists in main memory.

| Valid bit | Dirty bit | Reference bit | Physical page number |
|-|-|-|
- Valid bit (Set/Unset bit): Indicates whether the corresponding page is in main memory.
- Dirty bit (Modified bit): Indicates whether the page has been modified, and配合回写策略，判断是否需要 write back to disk.
- Reference bit (Used bit): Used in conjunction with replacement algorithms.

The CPU accesses instructions:
- The page table base address register stores the starting address of the process's page table.
- According to the high bits of the virtual address, find the corresponding page table entry.
- If the valid bit is set, concatenate the physical page number and the virtual address position to form a real physical address.
- If the valid bit is not set, it indicates a page fault, which needs the operating system to handle.

### TLB (Translation Lookaside Buffer)
TLB is made up of cache memory, based on the principle of locality, the page table in memory is called slow table. When addressing, first check the fast table to see if there is a hit. If not, access the main memory page table.
The fast table typically adopts group associative or fully associative way.

- Tag (Valid bit) | Reference bit | Physical Page number |

## Segmented Virtual Memory
Dividing logical addresses into segment number and segment offset, virtual and real address translation are achieved by segment table. The segment table records the corresponding relationship between the logical segments in the program and their storage locations in main memory.

|||||
|-|-|-|-|
- Segment Number | Segment starting address | Set/Unset bit | Segment length |

The CPU accesses memory:
- According to the segment number, concatenate with the base address of the segment table to form a corresponding row in the segment table.
- According to the set/unset bit in the segment table, determine whether to load into main memory.
- If loaded into main memory, read out the starting address of the segment from the segment table, and add the segment offset to obtain the corresponding physical address in main memory.

## Segmented Paging Virtual Memory
According to the logical structure of the program, divide each segment into a fixed size page, and also divide the main memory space into equally sized pages. The program still uses page as the basic unit to transfer data between main memory and secondary storage.
Each program has one segment table, and each segment table corresponds to one page table. The length of each segment must be a multiple of the page length.

|||||
|-|-|-|
- Segment Number | Page number in segment | Offset within Page |

The CPU accesses memory:
- According to the segment number, obtain the address of the corresponding segment table.
- Get the starting address of the corresponding segment from the segment table and add the virtual address (segment page number) to form a page table address.
- Obtain the physical page number from the page table and combine it with the offset within the page to form a real address in main memory.
