# Linear Table
## I. Logical Structure and Basic Operations
### 1. Logical Structure
- A finite sequence of n data elements with the same data type, with a length of n, where n = 0 represents an empty table.
- Header: The first element
- Tail: The last element
- Except for the first element, each element has exactly one direct predecessor.
- Except for the last element, each element has exactly one direct successor.
### 2. Basic Operations
```c++
initList(&L);
len(L);
locateElem(L, i);
getElem(L, i);
listInsert(&L, i, e);
listDelete(&L, i, &e);
printList(L);
isEmptyList(L);
destroyList(&L);
```
## II. Sequential Storage Structure
### 1. Definition: Also known as an ordered table, it uses a set of addressable storage units to sequentially store the data elements of a linear table, making the logically adjacent two elements physically adjacent.
- The starting location of the storage space: `data[ ]`
- Maximum storage capacity for the ordered table: `MaxSize`
- Current maximum length of the ordered table: `len`
Characteristics:
- Random access, O(1) time complexity to access
- High storage density, each node only stores a data element
- No need to spend space establishing logical relationships between data elements, determined by the physical adjacency property
- Logically and physically adjacent, insertion and deletion operations require moving large numbers of elements.
### 2. Basic Operations
(1) Inserting an Element
```c++
//插入元素
boolean listInsert(SqList &L, int i, Elemtype e){
	if(i < 1 || i  L.len + 1)
		return -1;
	if(L.len = MaxSize)
		return -1;
	for(int j = L.len; j  i; j--)
		L.data[j] = L.data[j - 1];
	L.data[i - 1] = e;
	L.len++;
}
```
Analysis
- Best case: Inserting **(i=n+1)** at the end of the table, without moving any elements, resulting in a time complexity of O(1).
- Worst case: Inserting **(i=1)** at the beginning of the table, requiring n moves to shift all other elements, leading to a time complexity of O(n).
- Average case: Assuming $$P_i$$ (**$$P_i = \frac{1}{n+1}$$**) is the probability of inserting a node at position i, the average number of moves required to insert a node into a linear table of length n is $$\frac{n}{2}$$, resulting in a time complexity of O(n).

(2) Deleting elements
```c++
//删除元素
boolean listDelete(SqList &L, int i, Elemtype e){
	if(i < 1 || i  L.len + 1)
		return -1;
	e = L.data[i-1];
	for(int j = i; j < L.len; j++)
		L.data[j-1] = L.data[j];
	L.len--;
	return true;
}
```
Analysis
- Best case: Inserting **(i=n)** at the end of the table, without moving any elements, with a time complexity of O(1).
- Worst case: Inserting **(i=1)** at the beginning of the table, requiring n moves to shift elements, resulting in a time complexity of O(n).
- Average case: Assuming $$P_i$$ (**($$P_i = \frac{1}{n+1}$$)**) is the probability of inserting a node at position i, the average number of moves required when inserting a node into a linear table of length n is $$\frac{n-1}{2}$$, resulting in a time complexity of O(n).

(3) Searching for elements
```c++
	int locateElem(SqList &L, Elemtype e){
		int i;
		for(i = 0; i < L.len; i++)
			if(e == L.data[i])
				return i+1;
	}
```
Analysis

- Best case: The element being searched for is in the header, requiring only one comparison, with a time complexity of O(1).
- Worst case: The element being searched for is at the end or does not exist, requiring n comparisons, resulting in a time complexity of O(n).
- Average case: Assuming $$P_i$$ (**$$, which is $$\frac{1}{n}$$)** is the probability that a node is at position i, the average number of moves required to insert a node into a linear list of length n is $$\frac{n+1}{2}$$, resulting in a time complexity of O(n).

# Linked List Data Structure
## 1. Creating a Singly Linked List
### (1) Head Insertion
- Allocate memory for the new node
- Insert the new node at the head of the list (head node position)
- The order of nodes in the linked list is the reverse of the input order
### (2) Tail Insertion
- Allocate memory for the new node
- Insert the new node at the tail of the list (tail node position)
- The order of nodes in the linked list is consistent with the input order
## 2. Searching for a Node by Value
- Start from the first node and traverse downwards using the next pointer, until the i-th node is found, otherwise return the pointer to the last node's pointer field, which is NULL.
## 3. Searching for a Node by Index
- Starting from the first node of the linked list, locate the corresponding node by sequentially increasing the index from front to back, and return the value, while also checking if the index exceeds the bounds.
## 4. Insertion
- The insertion operation inserts a new node with value x into the i-th position of the singly linked list.
- First, check if the insertion position is valid.
- Find the predecessor node of the insertion position.
- Insert the node after the predecessor node.
```c++
	p = getElem(L, i-1)
	s-next = p-next;
	p-next = s;
```
## 5. Deletion
- Delete the i-th node in a singly linked list
- First, check if the insertion position is valid
- Find the predecessor node of the node to be deleted
- Then, delete its next node
```c++
	p = getElem(L, i-1)
	q = p-next;
	q-next = p-next;
	free(q);
```
# Doubly Linked List
- A doubly linked list has two pointers, `prior` and `next`, which point to the previous and next nodes respectively.
- Insertion operation
```c++
	s-next = p-next;
	p-next-prior = s;
	s-prior = p;
	p-next = s;
```
- Delete operation
```c++
	p-next = q-next;
	q-next-prior = p;
	free(q);
```
