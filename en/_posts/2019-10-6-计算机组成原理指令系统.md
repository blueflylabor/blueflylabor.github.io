# Instruction System
## Instruction System
Is the core of the instruction set architecture (ISA)
The ISA primarily includes:
- Instruction format
- Data types and formats
- Storage method of operands
- Number of registers, bit length, and numbering accessible by programs
- Size and addressing method of storage space
- Addressing method
- Control method for executing instructions
### Basic Format of Instructions
An instruction consists of an operation code and an address code field
|||
|-|-|
|Operation Code|Address Code|

Operation Code:
- Indicates the operation to be performed by the instruction
- Identifies the instruction
- Understands the function of the instruction
- Differentiates the composition and usage methods of operands and addresses

Address Code:
- Gives the address of the operand being operated on
- One or more addresses of operands used in the operation
- The address where the result is stored
- The entry address of the subroutine called
### Instruction Length
Instruction length refers to the number of binary codes contained in a single instruction
Instruction word length depends on:
- The length of the operation code
- The length of the operand address code
- The number of operand addresses

Single-word instructions: equal to the machine word length
Half-word instructions: half of the machine word length
Double-word instructions: twice the machine word length
Fixed-length instruction word structure: all instructions in a single instruction system have the same length

#### Zero-address Instruction: No Display Address

||
|-|
|OP|

- Instructions without operands
- Zero-address operation instructions are typically used in stack computers and involve two operands implicitly popping from the top and bottom of the stack, which are then processed, and the result is implicitly pushed back onto the stack

#### One-address Instruction:

|||
|-|-|
|OP|$$A_1$$|

OP($$A_1$$) $$\to$$ $$A_1$$
- Only has a destination operand, reads the operand address from $$A_1$$, performs OP operation, and stores the result back at the original address

(ACC)OP($$A_1$$) $$\to$$ ACC
- Implicit dual-operand instruction, reads the operand from the address $$A_1$$, can implicitly assign another operand to ACC, and the result is also stored in ACC
- If the instruction length is 32 bits, the operation code occupies 8 bits, one address field occupies 24 bits

#### Two-address Instruction
||||
|-|-|-|
|OP|$$A_1$$|$$A_2$$|

($$A_1$$)OP($$A_2$$) $$\to$$ $$A_1$$

- Commonly used arithmetic and logical operation instructions, require two operands, need to specify the destination operand and source operand respectively, the destination operand address is also used to store the result of the operation
- Instruction word length is 32 bits, the operation code occupies 8 bits, two address fields each occupy 12 bits, the direct addressing range of the instruction operand is $$2^{12}=4K$$

#### Three-address Instruction
|||||
|-|-|-|-|
|OP|$$A_1$$|$$A_2$$|$$A_3$$(Result)|

($$A_1$$)OP($$A_2$$) $$\to$$ $$A_3$$

- Instruction word length is 32 bits, the operation code occupies 8 bits, three address fields each occupy 8 bits, direct addressing range is $$2^8=256$$, the address field is the main memory address, so it takes 4 accesses to complete one instruction
- Takes one access to fetch an instruction. Two accesses to fetch operands, and one access to store the result

#### Four-address Instruction

||||||
|-|-|-|-|-|
|OP|$$A_1$$|$$A_2$$|$$A_3$$|$$A_4$$|

($$A_1$$)OP($$A_2$$) $$\to$$ $$A_3$$ ，$$A_4$$ = Next executed instruction's address

- Address field is 32 bits, the operation code occupies 8 bits, four address fields each occupy 6 bits, direct addressing range is $$2^6=64$$

### Fixed-length Operation Code Instruction Format
The highest part of the instruction word is allocated to a fixed number of bits (fixed length) for the operation code.
A system with n bits can represent $$2^{n}$$ instructions

### Types of Operation in Instructions
- Data transmission
- Arithmetic and logical operations
- Bit shifting
- Branching
- Input/output

## Addressing Method
Determines the address of the operand and the next instruction to be executed, which can be divided into:
- Instruction addressing: finding the next instruction
(1) Sequential addressing: automatically forms the next instruction by PC+(1)
(2) Jump addressing: used to implement branching, the address of the next instruction is not given by PC automatically, but given by the method of calculating the address in the current instruction. Whether it can jump and the result depends on the state register and the operation number

- Data addressing: finds the data in the current instruction
- Usually a field in the instruction word indicates the addressing method

|Operation Code|Addressing characteristic|Form address A|

### Common Data Addressing Methods

#### Implicit Addressing
Does not give the operand address directly, but gives the implicit address in the instruction
- Advantage: simplifies the instruction word length
- Disadvantage: needs to add hardware to store operands or implicit addresses

#### Immediate Addressing
The address field of the instruction points to an immediate value (immediate number), which is represented in binary and is usually preceded by a "#" symbol.
- Advantage: the instruction does not need to access the main memory during execution, so it has the shortest execution time
- Disadvantage: A's bit length limits the range of immediate numbers

#### Direct Addressing
The address field of the instruction points directly to the operand, also known as direct addressing.
- Advantage: simple, only 1 access to main memory, no need to calculate the operand address
- Disadvantage: A's bit length determines the addressing range of the operand and the operand cannot be easily modified

#### Indirect Addressing
The address field of the instruction points to a storage unit address, which is the actual address of the operand.
- Advantage: can expand the addressing range (the number of bits in EA is greater than the number of bits in A), facilitates programming (indirect addressing can conveniently complete subroutine return)
- Disadvantage: slower access

#### Register Addressing
The operation code directly specifies the register number pointing to the operand, the operand is located in the register indicated by $$R_i$$
- Advantage: does not need to access main memory during instruction execution, only accesses registers, which makes the instruction word short and fast execution, supports vector/matrix operations
- Disadvantage: Registers are expensive and limited

#### Register Indirect Addressing
The address field of the instruction points to the address of the register $$R_i$$ pointing to the operand, also known as register indirect addressing.
- Advantage: faster than general indirect addressing
- Disadvantage: requires accessing main memory

#### Relative Addressing
The PC content plus the form address A in the instruction word forms the effective address of the operand EA = (PC) + A, A is a bit offset relative to PC, can be positive or negative, and represented in binary. The number of bits in A determines the addressing range
- The address of the operand is not fixed but varies with the value of PC, and the relative distance between the instruction address and the operand is a fixed value, which is convenient for programming
- JMP A, CPU retrieves one byte, automatically executes (PC)+1 $$\to$$ PC, if the address of the jump instruction is X, and it occupies 2B, after retrieving this instruction, PC is incremented by 2, (PC)=X+2, execute the instruction, automatically jumps to X+2+A

#### Base Addressing
The base register BR in the CPU, its content remains unchanged during program execution, and the form address A can be changed (as an offset)
- Base register faces the operating system, and its content is determined by the operating system or management program
- The form address can be used to represent the range of storage space

#### Indexed Addressing
The effective address EA is the sum of the 32-bit address field and the indexed address IX, EA=(IX)+A
- IX can use a dedicated register or a general register
- Index register faces users, and the content of the index register can be changed by users during program execution (as an offset)

#### Stack Addressing
Stack is a specific storage area in memory (or a group of dedicated registers), which follows the principle of Last In First Out (LIFO). The address of the stack field is given by a specific register called stack pointer (SP). It can be hard stack and soft stack.
