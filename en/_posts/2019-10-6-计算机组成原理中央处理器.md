# Central Processing Unit (CPU)
Controller: Responsible for coordinating and controlling the execution of program instructions by the computer's components: instruction fetching, instruction decoding, and instruction execution.
Arithmetic Logic Unit (ALU): Performs data processing.
- Instruction Control
- Operation Control
- Data Processing
- Timing Control
- Interrupt Handling

### Basic Structure
The CPU consists of two main parts: the control unit (CU) and the arithmetic logic unit (ALU).

### Components
The CU includes:
- Program Counter (PC)
- Instruction Register (IR)
- Instruction Decoder
- Memory Address Register
- Memory Data Register
- Clock System
- Microoperation Signal Generator

### Instruction Execution Process
### Instruction Cycle
The time it takes for the CPU to fetch and execute one instruction from main memory, typically represented by a certain number of machine cycles. Each machine cycle may be equal or unequal. A machine cycle contains a certain number of clock cycles (ticks). The number of clock cycles in each machine cycle can vary.
- Unconditional Jump Instruction: Does not require fetching data from memory during the execution phase, only includes fetch and analyze phases.
- Indirect Addressing Instruction: To fetch operands, first access 1 time, to obtain an effective address, then access memory to retrieve the operand, all of which involve indirect cycles, located between fetch and execute phases.
- The CPU uses interrupt mechanism to exchange information between the host (CPU) and I/O devices. Before each instruction ends, the CPU sends an interrupt query signal. If there is an interrupt request, the CPU enters the interrupt response phase (interrupt cycle).

|||||
|-|-|-|-|
|Fetch Cycle|Index Cycle|Execute Cycle|Interrupt Cycle|

Only the difference in access to memory affects the fetch cycle, index cycle, and execute cycle. The fetch cycle is used for fetching instructions, the index cycle is used for fetching effective addresses, and the execute cycle is used for fetching operands. The interrupt cycle is used to save the program breakpoint.

The CPU has 4 flags triggers (FE/IND/EX/INT)

### Instruction Cycle Data Flow
#### Fetch Cycle
According to PC, the instruction code is fetched from main memory and put into IR.
- PC $$\to$$ IR $$\to$$ AddrBus $$\to$$ Mem
- CU $$\to$$ DataBus $$\to$$ MDR $$\to$$ Mem
- Mem $$\to$$ DataBus $$\to$$ MDR $$\to$$ IR (stores the instruction)
- CU $$\stackrel{SignalControl::FE}{\longrightarrow}$$ [(PC) + 1 $$\to$$ PC]

#### Index Cycle
To fetch the effective address of the operand, for example: The address code in the instruction is sent to MAR and then sent to the address bus. The CU sends a read command to the memory to obtain the effective address and store it in MDR.
- Addr(IR) / MDR $$\to$$ MAR $$\to$$ AddrBus $$\to$$ Mem
- CU $$\stackrel{SignalI/O::READ}{\longrightarrow}$$ ControlBus $$\to$$ Mem (stores the effective address)
- Mem $$\to$$ DataBus $$\to$$ MDR

#### Execute Cycle
To fetch the operand, according to the operation code in IR, the ALU performs the operation to produce the execution result.
- No unified data flow direction

#### Interrupt Cycle
To handle interrupt requests. For example: If the program breakpoint is stored in stack, use SP to indicate the top of the stack, the stack operation first modifies the stack pointer and then stores the data.
- CU $$\stackrel{SignalStack::SP}{\longrightarrow}$$ [(SP) - 1 $$\to$$ SP] $$\to$$ MAR $$\to$$ AddrBus $$\to$$ Mem
- CU $$\stackrel{SignalI/O::WRITE}{\longrightarrow}$$ ControlBus $$\to$$ Mem
- PC $$\to$$ MDR $$\to$$ DataBus $$\to$$ Mem (stores the program breakpoint in main memory)
- CU $$\to$$ PC (interrupt service program entry address is sent to PC)

### Instruction Execution Scheme
- Single instruction cycle
- Multiple instruction cycles
- Pipeline scheme

## Data Path
### Function
Data path: The path through which data is transmitted between functional units. Controlled by the control unit, which generates control signals for different instructions to operate on the data path.
The function of the data path: to realize the data exchange between the CPU internal ALU and registers, and between registers.

### Basic Structure
- Single bus mode inside CPU
All input and output ends of the registers are connected to a single public bus. This structure is simple but results in frequent conflicts in data transmission, which reduces performance. When all components of the CPU have only one bus, it called single bus structure. If the CPU has two or more buses, it calls double bus structure or multi-bus structure
- Multi-bus mode inside CPU
All input and output ends of the registers are connected to multiple public buses. Compared with single bus structure, a single bus can transmit only one data at a time in one clock cycle, so the efficiency is very low. Therefore, in multi-bus mode, multiple buses can be used simultaneously for different data transmission, which improves efficiency
- Dedicated data path scheme
According to the direction of data and address flow during instruction execution process, connection lines are arranged to avoid using shared bus, which improve performance, but increase hardware complexity

#### Data transfer between registers
Through internal bus.
- (PC) $$\to$$ MAR, PCout and MARin effective

#### Data transfer between main memory and CPU
Data transfer between main memory and CPU needs to be realized by the internal bus of CPU.
- Read from Main Memory

- (PC) $$\to$$ MAR, PCout and MARin effective
- 1 $$\to$$ R, CU issues a read command
- MEM(MAR) $$\to$$ MDR, MDRin effective
- (MDR) $$\to$$ IR, MDRout and IRin effective
- CU $$\stackrel{SignalControl::DataFlow}{\longrightarrow}$$ [(PC) + 1 $$\to$$ PC]

#### Arithmetic or logical operation
Since ALU does not have internal storage function, it performs addition operation, the two numbers to be added must be valid in the two input outputs of ALU.

- (MDR) $$\to$$ MAR, MDRout and MARin effective
- 1  $$\to$$ R, CU reads a command
- MEM(MAR) $$\to$$ Operands read from main memory to MDR
- (MDR) $$\to$$ Y, MDRout and Yin effective
- (ACC) + (Y) $$\to$$ Z ACCout and ALUin effective
- (Z) $$\to$$ ACC, Zout and ACCin effective

- Microprogrammed controller

Two types of controllers:
- Hardwired controller
- Microprogrammed controller

### Hardwired Controller
- Generate various control signals based on the current instruction, time, and state.
- Composed of complex combinational logic gates and some flip-flops.

#### Hardwired Control Unit
The operation code in the instruction determines the output of different control signals (control signals) from the CU.
The input sources of the CU:
- Instruction generated by instruction decoding
- Machine cycle and tick signal generated by time system
- Feedback signal from execution unit
- Control signals from system bus (interrupt request, DMA request)

#### Hardwired Controller Timing System and Microprogram
- Clock Cycle
Use clock signal to generate ticks, each tick width corresponds to a clock cycle, the number of operations that can be performed in each tick is equal or not.
- Machine Cycle
Is the baseline time for executing one instruction, the access time to memory is fixed, so the machine cycle is usually defined as the shortest time to fetch a word from memory. Under the premise that the length of the storage word equals the length of the instruction word, the fetch cycle is considered as the machine cycle.
- Instruction Cycle

#### Microoperation Analysis
The control unit has the function of generating various operation control signals, which are used to operate the specified action.

The process of executing one instruction is divided into three working cycles: Fetch cycle, index cycle and execute cycle.

Fetch cycle
- (PC) $$\to$$ MAR
- 1 $$\to$$ R
- M(MAR) $$\to$$ MDR
- (MDR) $$\to$$ IR
- OP(IR) $$\to$$ CU
- (PC) + 1  $$\to$$ PC

Index Cycle
- Addr(IR) $$\to$$ MAR
- 1  $$\to$$ R
- M(MAR)  $$\to$$ MDR

Execute Cycle
- Non-memory access instruction
- Memory access instruction

Interrupt Cycle
- Handle interrupt request. For example: If the program breakpoint is stored in stack, use SP to indicate the top of the stack, the stack operation first modifies the stack pointer and then stores the data.
- CU $$\stackrel{SignalStack::SP}{\longrightarrow}$$ [(SP) - 1 $$\to$$ SP] $$\to$$ MAR $$\to$$ AddrBus $$\to$$ Mem
- CU $$\stackrel{SignalI/O::WRITE}{\longrightarrow}$$ ControlBus $$\to$$ Mem
- PC $$\to$$ MDR $$\to$$ DataBus $$\to$$ Mem (stores the program breakpoint in main memory)
- CU $$\to$$ PC (interrupt service program entry address is sent to PC)

#### CPU Control Mode
- Synchronous control mode
All control signals come from a unified clock signal.
- Asynchronous control mode
There is no baseline time signal, each part works according to its own speed, and connects by responding way.
- Combined control mode
Most of them adopt synchronous control, but some adopt asynchronous control

The input sources of the CU:
- Instruction generated by instruction decoding
- Machine cycle and tick signal generated by time system
- Feedback signal from execution unit
- Control signals from system bus (interrupt request, DMA request)

#### Hardwired Controller Timing System and Microprogram
- Clock Cycle
Use clock signal to generate ticks, each tick width corresponds to a clock cycle, the number of operations that can be performed in each tick is equal or not.
- Machine Cycle
Is the baseline time for executing one instruction, the access time to memory is fixed, so the machine cycle is usually defined as the shortest time to fetch a word from memory. Under the premise that the length of the storage word equals the length of the instruction word, the fetch cycle is considered as the machine cycle.
- Instruction Cycle
